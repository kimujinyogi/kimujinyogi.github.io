## ファイル・ディレクトリ規則

### 新規記事のパス構造

```
campcar/content/posts/<記事ディレクトリ名>/index.md
```

### ファイル命名規則

既存の記事から読み取れるパターン：

| プレフィックス | カテゴリ | 例 |
|---|---|---|
| `z` | 雑談・日常 | `z20260319.md` |
| `b` | 本のレビュー | `b20250507.md` |
| `mc` | マインクラフト | `mc20210725.md` |
| `campcar_` | キャンピングカー | `campcar_20200806.md` |
| `parenting_` | 育児 | `parenting_20240819.md` |
| `dev` | 開発 | `dev20240228.md` |
| `gm` | ゲーム | `gm20230606.md` |
| `ik` | 育児記 | `ik20230730.md` |
| `st` | 雑談/その他 | `st20201203.md` |

日付フォーマット: `YYYYMMDD`（今日の日付を使う）

同じ日に複数記事がある場合は末尾に `_0`, `_1` などを付ける。

## ショートコード

### 画像（中央揃え）

```
{{< imagelink src="/記事ディレクトリ名/image.jpg" class="imagecenter" title="キャプション" >}}
```

### サムネイル付き画像（拡大プレビュー）

```
{{< thumblink src="/記事ディレクトリ名/image.jpg" thumbnail="/記事ディレクトリ名/image_t.jpg" class="imagecenter" title="キャプション" >}}
```

- `src`: フル解像度画像
- `thumbnail`: サムネイル（`_t.jpg` または `_b.jpg` サフィックスが多い）

```
{{< youtube VIDEO_ID >}}
```

### 画像ファイルの配置場所

```
campcar/static/<記事ディレクトリ名>/image.jpg
```

## 記事作成の手順

ユーザーから記事作成を依頼されたら、以下の手順で進める：

1. **情報収集**: タイトル、カテゴリ、内容の概要をユーザーに確認する（不明な場合）
2. **ファイル名決定**: 上記の命名規則に従って `<ディレクトリ名>` を決定する
3. **Front matter 生成**: 適切な category, tags, series を設定する
4. **本文作成**: 下記の文体ガイドに従って日本語で記事を書く
5. **ファイル作成**: `campcar/content/posts/<ディレクトリ名>/index.md` に書き込む

## 文章スタイルガイド（最重要）

著者・金珍奕の文体を正確に再現するためのガイドです。

### 基本口調

- **です・ます調**（敬体）が基本。ただし砕けた表現も自然に混在する
- 話し言葉に近い、友人に話しかけるような自然な日本語
- 「〜ですね。」で感想・共感を表現する
- 「〜と思います」で意見を柔らかく伝える

### 締め言葉（最重要・必ず使う）

**記事の末尾は必ず「では、また！」で締める。**
感嘆符の数は気分で変わる（「では、また！」「では、また！！」など）。
この締め言葉は絶対に省略しない。

### 笑いの表現

- `w` `ｗ` `笑` を使う（絵文字は使わない）
- 例: 「これは失敗でしたｗ」「我ながら天才ですね（笑）」

### 驚き・落胆の表現

「・・・」「・・・・・」で段階的に表現する：
- 軽い驚き: 「あ・・」
- 大きな失敗・落胆: 「あ・・・・・」「やってしまった・・・」
- 困惑・迷い: 「んーなんでしょう・・・」

### よく使うフレーズ

- 「なんだかんだ」（結局こうなった、という流れで）
- 「気を取り直して」（失敗・問題の後に再挑戦するとき）
- 「正直に言って」（本音を語るとき）
- 「めっちゃ」「すごい」（強調表現）
- 「あるあるですね」（共感の表現）

### 感情表現（具体例）

| 感情 | 表現例 |
|---|---|
| 喜び・感激 | 「嬉しくて嬉しくて」「とても満足！！」「ニヤニヤです」 |
| 驚き・失敗 | 「あ・・」「あ・・・・・」「やってしまった・・・」 |
| 困惑 | 「んーなんでしょう・・・」 |
| 自己反省 | 「心に命じよ、いかなる時でも油断は禁物」 |

### 視覚的強調

- 二重引用符「」や<<>>で重要な言葉・固有名詞を囲む
- 擬音語を活用する（例: 「ががががっ」）

### NG パターン（絶対に使わない）

- 絵文字（😊🎉 など）
- 硬い文語体・論文調
- 過度に丁寧すぎるビジネス文書的表現
- 「〜でございます」「〜いたします」

## 記事構成パターン

1. **導入**: 状況・日付の説明でさらっと始める
2. **本文**: 見出し（`##` や `###`）で話題を区切る
3. **画像**: 要所に挿入して視覚的に補う
4. **体験談**: 失敗談・試行錯誤を率直に語る（失敗も隠さない）
5. **まとめ**（任意）: 「最後に」で結論・感想をまとめる
6. **締め**: **「では、また！」**（必須）

### 家族中心の視点

娘・息子・妻への言及が頻繁に登場する：
- 家族の反応・気持ちを詳細に描写する
- 親としての試行錯誤を正直に記録する
- 子供たちの成長や喜びを大切に書く

## 注意事項

- `themes/zzo/` は Git サブモジュールなので直接編集しない
- CSS のカスタマイズは `campcar/assets/css/custom.css` で行う
- テンプレートのオーバーライドは `campcar/layouts/` 以下に配置
- `master` ブランチは CircleCI が自動生成するので直接編集しない
- `campcar` ブランチへの push でデプロイが走る
